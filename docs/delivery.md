# 交付流程

## 1. 先沟通需求

开始前先确认这些信息：

- 你当前的设备和网络环境
- 你希望解决的问题
- 你要的是新搭建、修复还是长期维护
- 是否可以远程协助

## 2. 判断处理方式

通常会分成三类：

- 从零搭建
- 现网修复
- 长期维护

不同类型会决定后面的处理节奏和交付方式。

## 3. 开始实施

实施阶段通常包括：

- 环境检查
- 配置调整
- 规则整理
- 功能验证
- 必要的回滚预案保留

## 4. 交付说明

处理完成后，会尽量把这些内容讲清楚：

- 当前网络结构大概是什么样
- 改了哪些关键设置
- 后续使用要注意什么
- 哪些部分建议不要随便改

## 5. 后续维护

如果后续还会持续调整，可以继续做：

- 新设备接入
- 网络策略更新
- 线路问题排查
- 配置迁移
- 周期性维护
